# product_spec.md — 우리 둘의 캘린더

## 한 줄 정의
애인과 서로의 스케줄을 공유하고, 함께하는 일정·기념일·디데이를 챙기며, 날짜마다 사진과 메모로 소소한 일상을 남기는 — 펼치고 싶은 다이어리 같은 둘만의 캘린더.

---

## 분위기와 말투
- **무드:** 따뜻하고 포근한 파스텔 톤. 크림빛 종이 질감의 배경 위에 부드러운 코랄·살구·연한 세이지가 포인트. 캘린더인데 사무적이지 않고, 다이어리를 펼친 것 같은 다정함.
- **타이포:** 제목·날짜·짧은 문구에는 손글씨 느낌 서체, 본문/일정 텍스트는 읽기 편한 둥근 산세리프. 손글씨는 강조에만, 남발하지 않기.
- **말투:** 앱이 말을 걸 때는 반말의 다정한 톤. "오늘은 무슨 일이 있었어?", "우리 만난 지 D+124일", "아직 비어 있는 하루네 — 뭐라도 남겨볼까?" 같은 식. 명령형·기계적 안내문("항목을 입력하세요") 금지.
- **모션:** 사진을 붙이면 살짝 눕혀진 폴라로이드처럼 얹히고, 하트를 누르면 통통 튀는 정도의 작은 애정 표현. 과하지 않게.

---

## 들어갈 내용 (기능 범위)

### 경험 기능 (매일 만지는 것)
1. **각자 스케줄 공유** — 나의 일정, 애인의 일정을 각각 등록하고 한 화면에서 함께 본다. "나 / 너 / 우리" 세 가지를 색으로 구분.
2. **함께하는 일정** — 데이트, 여행, 약속 등 둘 공동 일정. '우리' 색으로 표시.
3. **기념일 · 디데이** — 사귄 날, 첫 여행 같은 기념일 등록. 상단에 D+/D- 위젯으로 항상 보이게. 매년 반복되는 기념일은 자동으로 다시 챙김.
4. **소소한 일상 기록** — 특정 날짜에 짧은 메모/한 줄 일기를 남긴다. (한 일, 기분 등)
5. **사진 붙이기** — 일정이나 날짜에 사진을 매달아, 캘린더 자체가 추억 앨범이 되도록. 날짜 칸에 작은 썸네일.
6. **추억 모아보기** — 사진이 붙은 날들을 앨범/타임라인처럼 한데 모아 되돌아보는 뷰.
7. **서로에게 남기는 메모(쪽지)** — 날짜에 매인 기록과 별개로, 상대에게 남기는 짧은 쪽지.

### 기반 기능 (경험을 지탱하는 것)
8. **로그인 · 커플 연결** — 각자 로그인하고, 초대코드로 둘을 하나의 공간으로 잇는다. 세상에서 오직 둘만 들어오는 비밀 공간이라는 정체성의 뿌리.
9. **클라우드 저장** — 일정·메모·쪽지·사진을 클라우드에 저장·동기화. 기기를 바꾸거나 앱을 다시 깔아도 로그인만 하면 추억이 그대로. (Between의 데이터 소실 사고를 반복하지 않기 위한 안전장치)
10. **실시간 동기화** — 한 사람이 일정·메모를 추가/수정하면 상대 화면에도 즉시 반영. "화요일에 시간 돼?"를 매번 묻지 않게 하는 핵심.

> ※ **외부 캘린더 연동(구글/애플 가져오기)은 이번 범위에서 제외.** 너희만의 다정한 무드가 외부 일정으로 희석되지 않도록, 일정은 이 공간 안에서만 쌓는다.
> ※ **AI(자연어 입력·문구 추천)도 이번 범위에서 제외.** (사용자 결정 — 담백함 유지)
> ※ 채팅·통화·위치추적·할일/쇼핑리스트·소셜 피드·결제·광고도 넣지 않는다. (레퍼런스에서 "무거워지는 원인"으로 지목된 것들)

## 화면 목록

### 1. 메인 — 월간 캘린더
- 상단: 디데이 위젯("우리 만난 지 D+○○일") + 다가오는 기념일 하나.
- 본문: 월간 격자. 각 날짜 칸에 일정 점(너/나/우리 색), 사진이 있는 날은 작은 썸네일.
- 하단/플로팅: 일정 추가 버튼(+).
- **실시간:** 애인이 방금 추가·수정한 일정이 새로고침 없이 스르륵 나타난다. 방금 바뀐 항목은 잠깐 은은하게 반짝여 "상대가 남긴 것"임을 알려줌.
- **빈 화면:** 이제 막 시작해 일정이 하나도 없을 때 — "아직 텅 빈 달력이야. 첫 일정을 남겨볼까?" + 큰 추가 버튼. 삭막한 빈 격자만 보여주지 않기.
- **로딩:** 종이 위에 점이 통통 떠오르는 잔잔한 인디케이터. 골격(스켈레톤)으로 격자 윤곽만 먼저.
- **에러:** "달력을 불러오지 못했어. 잠깐 뒤에 다시 열어볼까?" + 다시 시도 버튼. (기록이 사라진 게 아니라 못 불러온 것임을 분명히 안심시키기)

### 2. 날짜 상세
- 그 날의 일정(너/나/우리 구분), 메모/한 줄 일기, 붙은 사진들을 한 화면에.
- 이 날에 일정·메모·사진 추가 가능.
- **빈 화면:** "이 날은 아직 조용하네. 무슨 일이 있었어?" + 일정/메모/사진 추가 3가지 입구.
- **로딩:** 상단 날짜만 먼저 뜨고 아래 내용은 스켈레톤.
- **에러:** "이 날의 기록을 여는 데 문제가 있었어." + 다시 시도.

### 3. 일정 추가 / 편집
- 입력: 제목, 날짜(+시간, 선택), 주인(너/나/우리), 색, 메모, 반복 여부, 사진 첨부.
- **빈 화면:** 해당 없음(항상 폼).
- **로딩:** 저장 시 버튼이 잔잔한 로딩 상태로. 저장 성공하면 작은 하트 애니메이션.
- **에러:** "저장이 안 됐어. 방금 쓴 내용은 그대로 남겨뒀으니 다시 눌러줘." — **입력 내용 유실 없이** 재시도.

### 4. 기념일 · 디데이
- 등록한 기념일 목록 + 각각의 D-day 카운트다운, 매년 반복 옵션.
- **빈 화면:** "특별한 날을 하나 정해볼까? 처음 만난 날부터 시작하면 좋아." + 추가 버튼.
- **로딩:** 카드 스켈레톤.
- **에러:** "기념일을 불러오지 못했어." + 다시 시도.

### 5. 추억 모아보기 (앨범/타임라인)
- 사진이 붙은 날들을 최신순 또는 월별로 모아 보여줌. 각 사진 → 그 날짜 상세로 이동.
- **빈 화면:** "아직 모인 추억이 없어. 오늘 사진 한 장부터 붙여볼까?" + 사진 추가로 유도.
- **로딩:** 사진 자리를 흐린 블록으로 먼저 채우고 순차 로드.
- **에러:** "사진을 불러오는 중 문제가 생겼어." + 다시 시도. (사진이 삭제된 게 아님을 명시)

### 6. 쪽지 (서로에게 남기는 메모)
- 상대에게 남긴/받은 짧은 쪽지 목록.
- **빈 화면:** "첫 쪽지를 남겨봐. 상대가 캘린더를 열면 살포시 보일 거야."
- **로딩:** 쪽지 카드 스켈레톤.
- **에러:** "쪽지를 불러오지 못했어." + 다시 시도.

### 0. 로그인 · 커플 연결 (첫 진입)
- **로그인:** 각자 계정으로 로그인. 이미 연결된 커플이면 바로 우리 공간으로.
- **커플 연결:** 처음이면 초대코드를 만들어 상대에게 주거나, 받은 코드를 입력해 둘을 하나의 공간으로 잇는다. 연결 시 사귄 날(디데이 기준)을 함께 입력.
- **빈 화면(연결 대기):** 코드는 만들었지만 상대가 아직 안 들어온 상태 — "상대가 코드를 넣으면 여기서 둘이 이어져. 조금만 기다려줄까?" + 코드 다시 보내기.
- **로딩:** 로그인·연결 확인 중 잔잔한 인디케이터.
- **에러:** 로그인 실패 시 "로그인이 안 됐어. 다시 시도해줘." / 연결 실패 시 "연결이 안 됐어. 코드를 다시 확인해줘." — **상대 데이터에 영향 없음**을 보장.
- *(보안 원칙: 비밀번호·인증정보는 이 문서 범위에서 직접 다루지 않고, 로그인 제공자에 위임한다.)*

## 데이터 모델
지금 기획엔 저장할 데이터가 분명히 있어서 모델을 정의해둔다. 모든 데이터는 **클라우드에 저장·동기화**되며, 변경은 **실시간으로 상대에게 전파**된다. 그래서 대부분의 레코드에 `updatedAt`(최근 변경 시각), `updatedBy`(누가 바꿨는지)를 두어 실시간 반영과 "방금 상대가 남긴 것" 표시에 쓴다. (아래는 구조 설계용 스키마)

**Couple (둘의 공간)**
- `id`
- `memberA`, `memberB` — 두 사람 참조
- `inviteCode` — 커플 연결용 초대코드
- `startDate` — 사귄 날 (디데이 기준)
- `theme` — 파스텔 색 테마 설정

**Member (사용자)**
- `id`
- `authId` — 로그인 제공자의 사용자 식별자 (비밀번호 자체는 저장하지 않음)
- `displayName` — 표시 이름
- `color` — 캘린더에서 이 사람 일정 색 (기본: 나=코랄, 너=세이지)

**Event (일정)**
- `id`
- `title`
- `date` (+ `startTime`, `endTime` 선택)
- `owner` — `me` | `you` | `us`
- `memo` — 일정에 붙는 메모 (선택)
- `repeat` — 없음/매주/매월/매년 (선택)
- `photos[]` — 첨부 사진 참조 (선택)
- `createdBy` — 누가 만들었는지

**Anniversary (기념일·디데이)**
- `id`
- `title` — 예: "처음 만난 날", "첫 여행"
- `date`
- `repeatYearly` — 매년 반복 여부
- `pinnedToWidget` — 상단 위젯에 고정할지

**Note (날짜 기록 / 한 줄 일기)**
- `id`
- `date` — 매인 날짜
- `content`
- `author` — `me` | `you`
- `photos[]` — 선택

**Message (쪽지)**
- `id`
- `content`
- `from`, `to`
- `createdAt`
- `readAt` — 읽음 표시 (선택)

**Photo (사진)**
- `id`
- `storagePath` — 클라우드 파일 저장소의 경로 (원본은 클라우드에 보관)
- `url` — 표시용 주소
- `caption` — 선택
- `date` — 찍은/기록한 날짜
- `attachedTo` — `event` | `note` | `date` (무엇에 매였는지)

> 저장 원칙: 업로드 시 이미지를 적당히 리사이즈/압축해 무료 저장 용량 안에서 오래 쓰도록 한다. 소중한 사진인 만큼 정기 백업(또는 내려받기) 습관을 함께 둔다.

## "이 페이지가 완성됐다고 느껴질 순간"
아침에 페이지를 열었을 때,
1. 상단에서 **오늘 우리가 D+며칠인지, 다음 기념일이 며칠 남았는지** 바로 보이고,
2. 오늘 격자에서 **너의 일정과 나의 일정이 색으로 나란히** 한눈에 들어오고,
3. 어제 날짜를 눌러 **사진 한 장과 한 줄 메모를 세 번의 탭 안에** 남길 수 있고,
4. **애인이 방금 저쪽에서 추가한 일정이 내 화면에 스르륵 나타나** "아 오늘 이거 있었지" 하게 되고,
5. 며칠 뒤 다른 기기에서 로그인해도 **추억이 하나도 빠짐없이 그대로** 있고, '추억 모아보기'를 열면 **그 사진들이 앨범처럼 쌓여 스크롤하며 미소 짓게** 된다면 —

그때 이 페이지는 "일정 관리 도구"가 아니라 **매일 열고 싶은 둘만의 다이어리**가 된 거고, 완성이라고 느껴질 거야.

## 추가하면 좋을 기능 제안 (반영 여부는 네가 결정)
클라우드 저장·로그인·실시간은 논의 끝에 **본문 기능(8~10번)으로 정식 편입**했어. 여기 남은 건 이번엔 넣지 않기로 한 것들이야.

- **AI (제외):** "내일 저녁 7시 영화" 같은 자연어 일정 입력, 기념일 문구 추천을 검토했지만, 담백함을 지키려고 이번 범위에서 뺐어. 나중에 입력이 번거롭게 느껴지면 자연어 입력만 가볍게 붙이는 걸 추천.
- **외부 캘린더 연동 API (보류):** 구글/애플 캘린더를 가져와 기존 일정과 겹쳐보기. 편의는 크지만, 너희만의 다정한 무드가 외부 일정으로 희석될 수 있어 이번 범위에선 뺐어. 나중에 "회사 일정도 한눈에 보고 싶다"가 절실해지면 그때 붙이는 걸 추천.
- **결제·광고 (제외):** 개인적인 둘만의 공간이라 수익화 요소는 무드를 해쳐서 넣지 않아.

> 참고: 편입한 기능은 대부분 Supabase·Firebase 같은 서비스의 **무료 플랜 안에서 구현 가능**해. 시작은 무료로, 사진이 아주 많이 쌓이거나 자동 백업이 꼭 필요해질 때 유료를 고민하면 돼.
